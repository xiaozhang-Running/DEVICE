# 设备仓库管理系统 - 对话框概要设计文档

## 1. 系统概述

设备仓库管理系统是一个基于.NET 10.0和React开发的企业级仓库管理解决方案，旨在帮助企业高效管理设备、耗材和原材料的入库、出库、库存跟踪等业务流程。系统采用前后端分离架构，前端使用React和Ant Design组件库实现，后端使用.NET 10.0和Entity Framework Core。

### 1.1 系统目标

- 提供直观、高效的仓库管理界面
- 实现设备、耗材、原材料的全生命周期管理
- 支持项目出入库的完整流程
- 提供实时库存监控和预警
- 实现用户权限的精细控制
- 提供详细的操作日志和报表

## 2. 对话框组件架构

### 2.1 技术栈

- **前端框架**：React 18.2.0
- **UI组件库**：Ant Design 5.12.8
- **状态管理**：React useState, useEffect, useCallback
- **HTTP请求**：Axios 1.6.2
- **时间处理**：Dayjs 1.11.10
- **构建工具**：Vite 5.0.8

### 2.2 对话框类型

系统中的对话框主要分为以下几类：

1. **表单对话框**：用于创建、编辑数据，包含完整的表单验证
2. **确认对话框**：用于确认危险操作，如删除、提交等
3. **预览对话框**：用于查看详细信息，只读模式
4. **选择对话框**：用于从列表中选择物品、用户等
5. **详情对话框**：用于查看具体项目的详细信息

### 2.3 对话框设计原则

- **一致性**：所有对话框使用统一的设计风格和交互模式
- **简洁性**：只包含必要的字段和操作，避免信息过载
- **响应式**：适配不同屏幕尺寸
- **可访问性**：支持键盘导航和屏幕阅读器
- **错误处理**：提供清晰的错误提示和处理机制

## 3. 对话框详细设计

### 3.1 项目出库管理对话框

#### 3.1.1 新建/编辑项目出库单对话框

**功能描述**：用于创建新的项目出库单或编辑现有出库单，是系统中最复杂的对话框之一

**UI组件**：
- Modal (Ant Design) - 宽度1200px，居中显示
- Form (Ant Design) - 垂直布局
- Input, Select, DatePicker, Upload, TextArea
- Card, Table, Tag, Button

**主要字段**：

| 字段名 | 类型 | 验证规则 | 描述 | 默认值 |
|-------|------|---------|------|--------|
| outboundNumber | Input | 禁用 | 出库单号（自动生成） | 系统自动生成，格式：XMCK+年月日+序号 |
| outboundType | Select | 必填 | 出库类型 | 无，必须选择 |
| logisticsMethod | Select | 可选 | 物流方式 | 无 |
| projectName | Input | 必填，最大长度200 | 项目名称 | 无 |
| projectManager | Input | 必填，最大长度200 | 项目负责人 | 无 |
| contactPhone | Input | 必填，手机号格式 | 联系电话 | 无 |
| projectTime | Input | 必填 | 项目时间，如：2026.03.01-2026.03.15 | 无 |
| returnDate | DatePicker | 可选，必须大于当前日期 | 预计归还日期 | 无 |
| usageLocation | Input | 可选，最大长度200 | 使用地 | 无 |
| outboundImages | Upload | 可选，最多5张，单张不超过5MB | 出库图片 | 空数组 |
| recipient | Input | 必填，最大长度200 | 领用人 | 无 |
| handler | Input | 必填，最大长度100 | 经办人 | 无 |
| warehouseKeeper | Input | 可选，最大长度100 | 库管员 | 无 |
| remark | TextArea | 可选，最大长度500 | 备注 | 无 |
| selectedItems | Table | 必填，至少一项 | 已选物品列表 | 空数组 |

**操作流程**：
1. 点击"新建项目出库"按钮打开对话框
2. 系统自动生成出库单号并填充
3. 选择出库类型和物流方式
4. 填写项目基本信息（项目名称、负责人、联系电话等）
5. 点击"选择物品"按钮打开物品选择对话框
6. 在物品选择对话框中选择物品类型，搜索并选择物品
7. 在物品详情对话框中查看物品详情并确认选择
8. 上传出库图片（可选）
9. 填写领用人、经办人和库管员信息
10. 点击"提交"按钮保存出库单
11. 系统验证表单，显示验证错误（如有）
12. 提交成功后显示成功提示，关闭对话框并刷新列表

**UI交互细节**：
- 表单字段实时验证，失去焦点时显示错误提示
- 物品选择对话框支持分页和搜索
- 已选物品列表支持数量编辑（仅耗材和原材料）
- 图片上传支持预览和删除
- 提交按钮在表单验证通过前禁用
- 对话框支持ESC键关闭

#### 3.1.2 物品选择对话框

**功能描述**：用于从可用物品列表中选择物品，支持按类型筛选和搜索

**UI组件**：
- Modal (Ant Design) - 宽度800px，居中显示
- Table, Pagination
- Input.Search, Button, Tag

**物品类型**：
- 专用设备（蓝色标签）- itemType: 1
- 通用设备（绿色标签）- itemType: 2
- 耗材（橙色标签）- itemType: 3
- 原材料（紫色标签）- itemType: 4

**表格字段**：
| 字段名 | 类型 | 描述 |
|-------|------|------|
| 物品类型 | Tag | 显示物品类型标签 |
| 物品名称 | 文本 | 物品名称 |
| 品牌 | 文本 | 物品品牌 |
| 型号 | 文本 | 物品型号 |
| 可用数量 | 文本 | 当前可用数量 |
| 操作 | Button | 选择按钮 |

**操作流程**：
1. 在新建/编辑出库单对话框中点击选择物品
2. 默认显示专用设备列表
3. 点击不同物品类型按钮切换物品类型
4. 在搜索框中输入关键词搜索物品
5. 点击物品行的"选择"按钮查看物品详情
6. 在物品详情对话框中确认选择
7. 点击"关闭"按钮返回出库单对话框

**UI交互细节**：
- 物品类型按钮点击后高亮显示
- 搜索支持防抖，300ms后自动搜索
- 分页支持快速跳转和页码显示
- 表格支持列排序
- 选择按钮hover时显示提示信息

| 型号 | 文本 | 物品型号 | 否 |
| 数量 | InputNumber | 选择数量 | 是（仅耗材和原材料） |

**UI交互细节**：
- 耗材和原材料的数量输入框支持上下调节
- 数量输入有最小值（1）和最大值（可用数量）限制
- 支持全选/取消全选功能
- 选择按钮在没有选择物品时禁用
- 表格支持列宽调整

## 6. 异常处理

### 6.1 错误提示

- **表单验证错误**：使用Ant Design Form的内置验证机制，在字段下方显示错误提示
- **API请求错误**：使用message.error显示友好的错误提示，如"获取数据失败，请稍后重试"
- **权限错误**：显示"权限不足，无法执行此操作"的提示
- **网络错误**：显示"网络连接失败，请检查网络设置"的提示
- **业务逻辑错误**：显示具体的业务错误信息，如"库存不足"、"设备已被占用"

### 6.2 边界情况

- **空数据**：当列表为空时，显示"暂无数据"的提示
- **加载失败**：当数据加载失败时，显示错误提示和重试按钮
- **提交失败**：当提交失败时，显示错误信息并保持对话框打开
- **超时处理**：设置API请求超时时间，超时后显示超时提示
- **重复提交**：在提交过程中禁用提交按钮，防止重复提交

### 6.3 错误处理流程

1. **前端错误捕获**：使用try-catch捕获前端代码错误
2. **API错误处理**：在axios拦截器中统一处理API错误
3. **错误日志**：记录错误信息到控制台，便于调试
4. **用户友好提示**：将技术错误转换为用户友好的提示信息
5. **错误恢复**：提供重试机制，允许用户在错误后继续操作

## 7. 性能优化

### 7.1 组件优化

- **使用React.memo**：对纯展示组件使用React.memo，避免不必要的重新渲染
- **使用useCallback**：缓存事件处理函数，避免每次渲染都创建新函数
- **使用useMemo**：缓存计算结果，避免重复计算
- **组件拆分**：将复杂对话框拆分为多个子组件，提高代码可维护性
- **虚拟滚动**：对于长列表使用虚拟滚动，减少DOM节点数量

### 7.2 数据加载优化

- **分页加载**：使用分页加载数据，避免一次性加载大量数据
- **懒加载**：对图片和非关键资源使用懒加载
- **缓存策略**：缓存已加载的数据，减少重复请求
- **防抖搜索**：对搜索输入使用防抖，减少API请求次数
- **批量请求**：合并多个API请求，减少网络开销

### 7.3 对话框性能

- **按需加载**：对话框内容按需加载，避免一次性渲染所有内容
- **条件渲染**：根据需要条件渲染对话框内容
- **图片优化**：对上传的图片进行压缩和预览
- **表单优化**：使用Form.Item的preserve属性，减少表单重新渲染
- **动画优化**：使用CSS动画代替JavaScript动画，提高性能

### 7.4 网络优化

- **请求合并**：合并多个相关请求
- **请求缓存**：缓存重复请求的结果
- **错误重试**：对网络错误进行自动重试
- **超时处理**：设置合理的请求超时时间
- **断点续传**：对大文件上传使用断点续传
