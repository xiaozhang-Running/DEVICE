# 设备仓库管理系统 - 对话框概要设计文档

## 1. 系统概述

设备仓库管理系统是一个基于.NET 10.0和React开发的企业级仓库管理解决方案，旨在帮助企业高效管理设备、耗材和原材料的入库、出库、库存跟踪等业务流程。系统采用前后端分离架构，前端使用React和Ant Design组件库实现，后端使用.NET 10.0和Entity Framework Core。

## 2. 对话框组件架构

### 2.1 技术栈

- **前端框架**：React 18.2.0
- **UI组件库**：Ant Design 5.12.8
- **状态管理**：React useState, useEffect
- **HTTP请求**：Axios
- **时间处理**：Dayjs

### 2.2 对话框类型

系统中的对话框主要分为以下几类：

1. **表单对话框**：用于创建、编辑数据
2. **确认对话框**：用于确认操作
3. **预览对话框**：用于查看详细信息
4. **选择对话框**：用于选择物品、用户等
5. **详情对话框**：用于查看详细信息

## 3. 对话框详细设计

### 3.1 项目出库管理对话框

#### 3.1.1 新建/编辑项目出库单对话框

**功能描述**：用于创建新的项目出库单或编辑现有出库单

**UI组件**：
- Modal (Ant Design)
- Form (Ant Design)
- Input, Select, DatePicker, Upload

**主要字段**：
| 字段名 | 类型 | 验证规则 | 描述 |
|-------|------|---------|------|
| outboundNumber | Input | 禁用 | 出库单号（自动生成） |
| outboundType | Select | 必填 | 出库类型（元动自用、对外租借、其他） |
| logisticsMethod | Select | 可选 | 物流方式（随身携带、跨越物流、德邦物流、顺丰物流、其他） |
| projectName | Input | 必填 | 项目名称 |
| projectManager | Input | 必填 | 项目负责人 |
| contactPhone | Input | 必填 | 联系电话 |
| projectTime | Input | 必填 | 项目时间 |
| returnDate | DatePicker | 可选 | 预计归还日期 |
| usageLocation | Input | 可选 | 使用地 |
| outboundImages | Upload | 可选 | 出库图片 |
| recipient | Input | 必填 | 领用人 |
| handler | Input | 必填 | 经办人 |
| warehouseKeeper | Input | 可选 | 库管员 |
| remark | TextArea | 可选 | 备注 |

**操作流程**：
1. 点击"新建项目出库"按钮打开对话框
2. 填写表单字段
3. 选择物品（通过物品选择对话框）
4. 上传出库图片
5. 点击"提交"按钮保存

#### 3.1.2 物品选择对话框

**功能描述**：用于从可用物品列表中选择物品

**UI组件**：
- Modal (Ant Design)
- Table, Pagination
- Input.Search, Button

**物品类型**：
- 专用设备（蓝色标签）
- 通用设备（绿色标签）
- 耗材（橙色标签）
- 原材料（紫色标签）

**操作流程**：
1. 在新建/编辑出库单对话框中点击选择物品
2. 选择物品类型（专用设备、通用设备、耗材、原材料）
3. 搜索物品
4. 点击"选择"按钮选择物品
5. 查看物品详情（通过物品详情对话框）

#### 3.1.3 物品详情对话框

**功能描述**：用于查看和选择具体物品的详细信息

**UI组件**：
- Modal (Ant Design)
- Table, InputNumber

**主要字段**：
| 字段名 | 类型 | 描述 |
|-------|------|------|
| 序号 | 文本 | 物品序号 |
| 物品名称 | 文本 | 物品名称 |
| 设备编号 | 文本 | 设备唯一编号 |
| 品牌 | 文本 | 物品品牌 |
| 型号 | 文本 | 物品型号 |
| 数量 | InputNumber | 选择数量（耗材和原材料可编辑） |
| 单位 | 文本 | 物品单位 |
| 设备状态 | 标签 | 设备状态（正常、损坏、报废） |
| 配件 | 文本 | 设备配件 |
| 备注 | 文本 | 备注信息 |

**操作流程**：
1. 在物品选择对话框中点击物品
2. 查看物品详情
3. 对于耗材和原材料，输入需要的数量
4. 点击"确认选择"按钮添加到已选物品列表

#### 3.1.4 预览对话框

**功能描述**：用于预览项目出库单的详细信息

**UI组件**：
- Modal (Ant Design)
- Card, Tag, Image

**显示内容**：
- 项目基本信息（项目名称、项目负责人、联系电话）
- 项目时间、预计归还日期、使用地
- 物品清单（序号、物品类型、物品名称、设备编号、品牌、型号、数量、单位、设备状态）
- 出库图片
- 领用人、经办人、库管员、备注

**操作流程**：
1. 在项目出库列表中点击"查看"按钮
2. 查看出库单详细信息
3. 点击"关闭"按钮关闭对话框

## 4. 对话框状态管理

### 4.1 状态变量

| 状态变量 | 类型 | 描述 |
|---------|------|------|
| modalVisible | boolean | 控制对话框显示/隐藏 |
| previewModalVisible | boolean | 控制预览对话框显示/隐藏 |
| selectItemModalVisible | boolean | 控制物品选择对话框显示/隐藏 |
| itemDetailModalVisible | boolean | 控制物品详情对话框显示/隐藏 |
| editingOutbound | object | 编辑中的出库单数据 |
| previewOutbound | object | 预览中的出库单数据 |
| form | FormInstance | 表单实例 |
| availableItems | array | 可用物品列表 |
| selectedItems | array | 已选物品列表 |
| outboundImages | array | 出库图片列表 |
| currentPage | number | 当前页码 |
| pageSize | number | 每页显示数量 |
| total | number | 总记录数 |
| itemPage | number | 物品列表当前页码 |
| itemTotal | number | 物品列表总记录数 |
| itemKeyword | string | 物品搜索关键词 |
| selectedItemType | number | 选择的物品类型 |
| currentItem | object | 当前选中的物品 |
| itemDetailList | array | 物品详情列表 |

### 4.2 数据流

1. **打开对话框**：设置对应的visible状态为true
2. **加载数据**：根据需要从API获取数据
3. **表单操作**：用户填写表单、选择物品等
4. **提交数据**：点击提交按钮，调用API保存数据
5. **关闭对话框**：设置对应的visible状态为false，重置相关状态

## 5. API接口设计

### 5.1 项目出库相关接口

| API路径 | 方法 | 功能描述 |
|---------|------|----------|
| /api/ProjectOutbounds | GET | 获取项目出库单列表 |

## 6. 异常处理

### 6.1 错误提示

- 使用Ant Design的message组件显示错误信息
- 对于API请求错误，显示友好的错误提示
- 对于表单验证错误，使用Form的内置验证机制

### 6.2 边界情况

- **网络错误**：显示网络连接失败提示
- **权限不足**：显示权限不足提示，禁用相关操作
- **数据不存在**：显示数据不存在提示
- **表单验证失败**：显示具体的验证错误信息

## 7. 性能优化

### 7.1 组件优化

- 使用React.memo优化组件渲染
- 使用useCallback缓存函数，避免不必要的重新渲染
- 使用debounce处理搜索输入，减少API请求次数

### 7.2 数据加载优化

- 分页加载数据，避免一次性加载大量数据
- 缓存已加载的数据，减少重复请求
- 使用懒加载处理图片，提高页面加载速度

### 7.3 对话框性能

- 对话框内容按需加载
- 大型表单使用虚拟滚动
- 图片上传使用预览模式，避免一次性上传多张图片

## 8. 安全考虑

### 8.1 数据验证

- 前端表单验证
- 后端API验证
- 防止SQL注入攻击

### 8.2 权限控制

- 使用usePermission hook检查用户权限
- 基于角色的权限控制
- 禁用无权限的操作按钮

### 8.3 数据安全

- 敏感数据加密传输
- 防止XSS攻击
- 防止CSRF攻击

## 10. 总结

本概要设计文档详细描述了设备仓库管理系统中的对话框组件设计，包括项目出库管理、项目入库管理、设备管理、耗材管理、原材料管理、用户管理、维修设备管理、库存管理和日志管理等模块的对话框。每个对话框都有明确的功能描述、UI组件、字段定义和操作流程。

系统采用React和Ant Design实现，具有良好的用户体验和性能表现。通过合理的状态管理和API设计，确保了系统的稳定性和可靠性。同时，系统还考虑了安全性和可测试性，为后续的开发和维护提供了良好的基础。
