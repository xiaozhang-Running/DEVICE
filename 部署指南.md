# 设备仓库管理系统 - 部署指南

## 1. 服务器环境准备

### 1.1 硬件要求
- **CPU**：至少 2 核
- **内存**：至少 4GB
- **存储空间**：至少 50GB
- **网络**：稳定的网络连接

### 1.2 软件要求
- **操作系统**：Windows Server 2019 或更高版本 / Ubuntu 20.04 LTS 或更高版本
- **.NET 运行时**：.NET 10.0 Runtime
- **数据库**：SQL Server 2019 或更高版本
- **Web 服务器**：
  - Windows：IIS
  - Linux：Nginx + Kestrel

## 2. 后端部署

### 2.1 发布后端应用

#### Windows 环境
1. **使用 Visual Studio 发布**：
   - 打开 `DeviceWarehouseSystem.sln` 解决方案
   - 右键点击 `DeviceWarehouse.API` 项目
   - 选择 **发布** → **发布到文件夹**
   - 选择目标文件夹（如 `D:\Publish\API`）
   - 点击 **发布**

2. **使用命令行发布**：
   ```bash
   cd DeviceWarehouseSystem\DeviceWarehouse.API
   dotnet publish -c Release -o D:\Publish\API
   ```

#### Linux 环境
```bash
cd DeviceWarehouseSystem/DeviceWarehouse.API
dotnet publish -c Release -o /var/www/api
```

### 2.2 配置数据库连接

1. **修改配置文件**：
   - 编辑 `appsettings.json` 文件
   - 更新 `ConnectionStrings` 部分：
     ```json
     "ConnectionStrings": {
       "DefaultConnection": "Server=<服务器地址>;Database=DeviceWarehouse;User Id=<用户名>;Password=<密码>;MultipleActiveResultSets=true"
     }
     ```

2. **初始化数据库**：
   - 运行应用时会自动创建数据库（如果不存在）
   - 系统会自动执行数据库迁移和种子数据初始化

### 2.3 部署到 IIS（Windows）

1. **安装 IIS**：
   - 打开 **服务器管理器** → **添加角色和功能**
   - 选择 **Web 服务器 (IIS)**
   - 安装所需的角色服务

2. **添加网站**：
   - 打开 **IIS 管理器**
   - 右键点击 **网站** → **添加网站**
   - 设置网站名称、物理路径（指向发布文件夹）和端口
   - 点击 **确定**

3. **配置应用池**：
   - 右键点击新创建的网站 → **高级设置**
   - 将 **应用池** 设置为 `.NET CLR 版本` 为 `无托管代码`
   - 确保应用池的 **身份** 有足够的权限访问数据库

4. **安装 ASP.NET Core 托管包**：
   - 下载并安装 [ASP.NET Core Runtime Hosting Bundle](https://dotnet.microsoft.com/en-us/download/dotnet/10.0)

### 2.4 部署到 Linux（使用 Systemd）

1. **创建服务文件**：
   ```bash
   sudo nano /etc/systemd/system/device-warehouse-api.service
   ```

2. **配置服务文件**：
   ```ini
   [Unit]
   Description=Device Warehouse API
   After=network.target

   [Service]
   WorkingDirectory=/var/www/api
   ExecStart=/usr/bin/dotnet DeviceWarehouse.API.dll
   Restart=always
   RestartSec=10
   SyslogIdentifier=device-warehouse-api
   User=www-data
   Environment=ASPNETCORE_ENVIRONMENT=Production
   Environment=DOTNET_PRINT_TELEMETRY_MESSAGE=false

   [Install]
   WantedBy=multi-user.target
   ```

3. **启动服务**：
   ```bash
   sudo systemctl daemon-reload
   sudo systemctl start device-warehouse-api
   sudo systemctl enable device-warehouse-api
   ```

## 3. 前端部署

### 3.1 构建前端应用

1. **安装依赖**：
   ```bash
   cd frontend
   npm install
   ```

2. **构建生产版本**：
   ```bash
   npm run build
   ```
   构建产物会生成在 `frontend/dist` 目录

### 3.2 部署到 Web 服务器

#### Windows IIS
1. **添加网站**：
   - 打开 **IIS 管理器**
   - 右键点击 **网站** → **添加网站**
   - 设置网站名称、物理路径（指向 `frontend/dist` 文件夹）和端口
   - 点击 **确定**

2. **配置静态文件处理**：
   - 确保 IIS 已安装 **静态内容** 功能
   - 配置 MIME 类型，确保 `.js`, `.css`, `.json` 等文件能正确处理

#### Linux Nginx
1. **安装 Nginx**：
   ```bash
   sudo apt update
   sudo apt install nginx
   ```

2. **配置 Nginx**：
   ```bash
   sudo nano /etc/nginx/sites-available/device-warehouse-frontend
   ```

3. **添加配置**：
   ```nginx
   server {
       listen 80;
       server_name example.com;

       root /var/www/frontend;
       index index.html;

       location / {
           try_files $uri $uri/ /index.html;
       }

       # 反向代理到后端 API
       location /api/ {
           proxy_pass http://localhost:5000/;
           proxy_http_version 1.1;
           proxy_set_header Upgrade $http_upgrade;
           proxy_set_header Connection keep-alive;
           proxy_set_header Host $host;
           proxy_cache_bypass $http_upgrade;
       }
   }
   ```

4. **启用配置**：
   ```bash
   sudo ln -s /etc/nginx/sites-available/device-warehouse-frontend /etc/nginx/sites-enabled/
   sudo nginx -t
   sudo systemctl restart nginx
   ```

### 3.3 配置前端 API 地址

1. **修改 API 基础地址**：
   - 编辑 `frontend/src/utils/request.js` 文件
   - 更新 `baseURL` 为后端 API 的部署地址：
     ```javascript
     const service = axios.create({
       baseURL: 'http://your-server-ip:5000/api',
       timeout: 10000
     });
     ```

2. **重新构建**：
   ```bash
   npm run build
   ```

## 4. 数据库部署

### 4.1 SQL Server 安装

#### Windows Server
1. **下载并安装 SQL Server**：
   - 从 [Microsoft 官网](https://www.microsoft.com/en-us/sql-server/sql-server-downloads) 下载 SQL Server 2019 或更高版本
   - 按照安装向导完成安装，选择 **数据库引擎服务**

2. **配置 SQL Server**：
   - 打开 **SQL Server 配置管理器**
   - 启用 TCP/IP 协议
   - 重启 SQL Server 服务

#### Linux
1. **安装 SQL Server**：
   ```bash
   wget -qO- https://packages.microsoft.com/keys/microsoft.asc | sudo apt-key add -
   sudo add-apt-repository "$(wget -qO- https://packages.microsoft.com/config/ubuntu/20.04/mssql-server-2019.list)"
   sudo apt update
   sudo apt install -y mssql-server
   sudo /opt/mssql/bin/mssql-conf setup
   ```

2. **安装 SQL Server 工具**：
   ```bash
   sudo apt install -y mssql-tools unixodbc-dev
   echo 'export PATH="$PATH:/opt/mssql-tools/bin"' >> ~/.bashrc
   source ~/.bashrc
   ```

### 4.2 数据库配置

1. **创建数据库**（如果自动创建失败）：
   ```sql
   CREATE DATABASE DeviceWarehouse;
   ```

2. **创建登录用户**：
   ```sql
   CREATE LOGIN WarehouseUser WITH PASSWORD = 'YourStrongPassword123!';
   CREATE USER WarehouseUser FOR LOGIN WarehouseUser;
   ALTER ROLE db_owner ADD MEMBER WarehouseUser;
   ```

3. **配置连接字符串**：
   - 更新后端 `appsettings.json` 中的连接字符串，使用创建的用户名和密码

### 5.1 启动服务

- **Windows IIS**：服务会自动启动
- **Linux Systemd**：
  ```bash
  sudo systemctl start device-warehouse-api
  ```

#### 后端日志

  ```bash
  sudo journalctl -u device-warehouse-api
  ```

#### 前端日志
- 查看浏览器开发者工具的 **Console** 和 **Network** 标签页

### 5.3 常见问题排查

1. **API 无法访问**：
   - 检查防火墙是否开放了对应端口
   - 检查服务是否运行
   - 检查数据库连接是否正确

2. **前端页面空白**：
   - 检查 Nginx/IIS 配置是否正确
   - 检查前端构建文件是否完整
   - 检查 API 地址是否配置正确

3. **数据库连接失败**：
   - 检查连接字符串是否正确
   - 检查 SQL Server 是否运行
   - 检查用户权限是否足够

## 6. 安全配置

### 6.1 HTTPS 配置

1. **获取 SSL 证书**：
   - 从证书颁发机构（CA）获取
   - 使用 Let's Encrypt 免费证书

2. **配置 HTTPS**：
   - **Windows IIS**：在网站绑定中添加 HTTPS 绑定
   - **Linux Nginx**：更新 Nginx 配置，添加 SSL 证书路径

### 6.2 防火墙设置

1. **Windows 防火墙**：
   - 打开 **Windows 防火墙高级安全**
   - 添加入站规则，允许 HTTP（80）和 HTTPS（443）端口

2. **Linux 防火墙**：
   ```bash
   sudo ufw allow 80
   sudo ufw allow 443
   sudo ufw allow 5000  # 后端 API 端口
   ```

### 6.3 安全最佳实践

- **定期更新**：定期更新 .NET 运行时和依赖库
- **备份数据库**：定期备份数据库
- **限制访问**：限制 API 访问来源
- **监控异常**：设置异常监控和告警

## 7. 部署验证

1. **访问前端**：
   - 打开浏览器，访问前端部署地址（如 `http://your-server-ip`）
   - 检查页面是否正常加载

2. **测试 API**：
   - 访问 API 文档地址（如 `http://your-server-ip:5000/swagger`）
   - 测试 API 接口是否正常响应

3. **验证数据库连接**：
   - 尝试创建一条测试数据，检查是否能成功保存到数据库

4. **测试完整流程**：
   - 测试项目出库、入库等核心业务流程
   - 验证所有功能是否正常工作

## 8. 自动化部署（可选）

### 8.1 使用 CI/CD 工具

1. **GitHub Actions**：
   - 创建 `.github/workflows/deploy.yml` 文件
   - 配置自动化构建和部署流程

2. **Jenkins**：
   - 安装 Jenkins
   - 创建构建任务，配置源码管理和构建步骤

### 8.2 脚本部署

1. **创建部署脚本**：
   ```bash
   # deploy.sh
   # 拉取代码
   git pull origin main
   
   # 构建后端
   cd DeviceWarehouseSystem/DeviceWarehouse.API
   dotnet publish -c Release -o /var/www/api
   
   # 构建前端
   cd ../../frontend
   npm install
   npm run build
   cp -r dist/* /var/www/frontend
   
   # 重启服务
   sudo systemctl restart device-warehouse-api
   sudo systemctl restart nginx
   ```

2. **运行部署脚本**：
   ```bash
   chmod +x deploy.sh
   ./deploy.sh
   ```
